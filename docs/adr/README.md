# Architecture Decision Records

This directory contains ADRs (Architecture Decision Records) for the `yali` project.
ADRs document significant technical decisions, their context, and the reasoning behind them.
The format follows [MADR (Markdown Architectural Decision Records)](https://adr.github.io/madr/).

## Adding a New ADR

1. Copy an existing ADR as a template.
2. Increment the number (e.g., `0003-...`).
3. Fill in the **Context**, **Decision**, **Consequences**, and **Alternatives Considered** sections.
4. Set **Status** to `Accepted` (or `Proposed` if still under discussion).
5. Add a row to the index table above.
6. Reference the new ADR from any relevant documentation.
